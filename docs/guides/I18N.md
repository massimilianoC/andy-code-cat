# Guide to the multilingual system (i18n) — ANDY

## Technology stack

| Package | Role |
|---|---|
| `react-i18next` | `useTranslation` hook and `I18nextProvider` integration |
| `i18next` | Core translation engine with the `translation` namespace |
| `i18next-browser-languagedetector` | Automatic browser language detection |

**Configuration file:** `apps/web/lib/i18n.ts`

---

## Architecture

```
apps/web/
├── lib/
│   └── i18n.ts              ← i18next configuration (detection, fallback)
├── i18n/
│   ├── en.json              ← English keys
│   └── it.json              ← Italian keys
└── components/
    ├── I18nProvider.tsx     ← root provider (wraps the app and updates <html lang>)
    └── LanguageSwitcher.tsx ← 🇮🇹 / 🇬🇧 toggle in the navbar and dashboard
```

### Language detection flow

1. Read `localStorage["andy_lang"]` if it was set by `LanguageSwitcher`
2. If it is missing, fall back to `navigator.language`
3. Default fallback language: `it`
4. Supported languages: `["it", "en"]`

## How to add a translated string

### 1. Add the key to both JSON files

**`apps/web/i18n/it.json`**

```json
{
  "myModule": {
    "title": "Titolo in italiano",
    "description": "Descrizione in italiano"
  }
}
```

**`apps/web/i18n/en.json`**

```json
{
  "myModule": {
    "title": "Title in English",
    "description": "Description in English"
  }
}
```

> **Rule:** the keys must be identical in both files. The namespace is always `translation` unless explicitly changed.

### 2. Use `useTranslation` inside the component

```tsx
"use client";

import { useTranslation } from "react-i18next";

export function MyComponent() {
    const { t } = useTranslation();

    return (
        <div>
            <h1>{t("myModule.title")}</h1>
            <p>{t("myModule.description")}</p>
        </div>
    );
}
```

> `useTranslation` is a React hook and should only be called inside React components or custom hooks.

## Dynamic value interpolation

To inject values into a string, use the `{{name}}` syntax:

**`en.json`**

```json
{ "welcome": "Hi, {{name}}! You have {{count}} projects." }
```

**In the component:**

```tsx
t("welcome", { name: user.name, count: projects.length })
// → "Hi, Mario! You have 3 projects."
```

---

## Pluralization

i18next handles plural forms with the `_one` and `_other` suffixes:

**`en.json`**

```json
{
  "notifications": {
    "running_one": "{{count}} running task",
    "running_other": "{{count}} running tasks"
  }
}
```

**In the component:**

```tsx
t("notifications.running", { count: runningCount })
// count=1 → "1 running task"
// count=3 → "3 running tasks"
```

## The `LanguageSwitcher` component

It is already included in the public navbar and in the dashboard. It calls `i18n.changeLanguage(lang)` and persists the choice in `localStorage["andy_lang"]`.

```tsx
import { LanguageSwitcher } from "@/components/LanguageSwitcher";

<LanguageSwitcher />
```

## Testing language switching locally

1. Start the dev or deploy-like stack (see `docs/guides/LOCAL_DOCKER_START.md`)
2. Open the app in the browser
3. Click 🇬🇧 in the navbar to switch the UI to English
4. Reload the page and verify the language is preserved through `localStorage["andy_lang"]`
5. Click 🇮🇹 to switch back to Italian

To force a language from the browser console:

```js
localStorage.setItem("andy_lang", "en");
location.reload();
```

## Adding a new language

1. Create `apps/web/i18n/<code>.json` such as `fr.json`
2. Add the code to `supportedLngs` inside `apps/web/lib/i18n.ts`
3. Add a new switch button in `LanguageSwitcher.tsx`
4. Rebuild the stack if needed
