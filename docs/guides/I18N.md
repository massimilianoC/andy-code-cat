# Guida al sistema multilingua (i18n) — ANDY

## Stack tecnologico

| Pacchetto | Ruolo |
|---|---|
| `react-i18next` | Hook `useTranslation`, provider `I18nextProvider` |
| `i18next` | Core engine, namespace `translation` |
| `i18next-browser-languagedetector` | Rilevamento automatico lingua |

**File di configurazione:** `apps/web/lib/i18n.ts`

---

## Architettura

```
apps/web/
├── lib/
│   └── i18n.ts              ← configurazione i18next (detection, fallback)
├── i18n/
│   ├── en.json              ← chiavi inglese
│   └── it.json              ← chiavi italiano
└── components/
    ├── I18nProvider.tsx      ← provider root (wrappa tutta l'app, aggiorna <html lang>)
    └── LanguageSwitcher.tsx  ← pulsanti 🇮🇹 / 🇬🇧 (navbar e dashboard)
```

### Flusso di rilevamento lingua

1. Legge `localStorage["andy_lang"]` (chiave custom impostata da `LanguageSwitcher`)
2. Se assente, usa `navigator.language`
3. Lingua di fallback: `it` (italiano)
4. Lingue supportate: `["it", "en"]`

## Come aggiungere una stringa tradotta

### 1. Aggiungere la chiave ai file JSON

**`apps/web/i18n/it.json`**

```json
{
  "mioModulo": {
    "titolo": "Titolo in italiano",
    "descrizione": "Descrizione in italiano"
  }
}
```

**`apps/web/i18n/en.json`**

```json
{
  "mioModulo": {
    "titolo": "Title in English",
    "descrizione": "Description in English"
  }
}
```

> **Regola:** le chiavi devono essere identiche in entrambi i file. Il namespace è sempre `translation` (default).

### 2. Usare `useTranslation` nel componente

```tsx
"use client";

import { useTranslation } from "react-i18next";

export function MioComponente() {
    const { t } = useTranslation();

    return (
        <div>
            <h1>{t("mioModulo.titolo")}</h1>
            <p>{t("mioModulo.descrizione")}</p>
        </div>
    );
}
```

> `useTranslation` è un hook React: chiamarlo solo dentro funzioni componente o hook custom.

## Interpolazione di valori dinamici

Per inserire variabili nella stringa, usa la sintassi `{{nome}}`:

**`it.json`**

```json
{ "benvenuto": "Ciao, {{nome}}! Hai {{count}} progetti." }
```

**Nel componente:**

```tsx
t("benvenuto", { nome: user.name, count: projects.length })
// → "Ciao, Mario! Hai 3 progetti."
```

## Pluralizzazione

i18next gestisce il plurale con il suffisso `_one` / `_other`:

**`it.json`**

```json
{
  "notifications": {
    "running_one": "{{count}} in corso",
    "running_other": "{{count}} in corso"
  }
}
```

**`en.json`**

```json
{
  "notifications": {
    "running_one": "{{count}} running",
    "running_other": "{{count}} running"
  }
}
```

**Nel componente:**

```tsx
t("notifications.running", { count: runningCount })
// count=1 → "1 running" (usa _one)
// count=3 → "3 running" (usa _other)
```

> Per l'italiano spesso `_one` e `_other` sono identici; per l'inglese divergono (e.g., "1 item" vs "2 items").

## Il componente `LanguageSwitcher`

Già incluso nella navbar pubblica e nella dashboard. Chiama `i18n.changeLanguage(lang)` e persiste la scelta in `localStorage["andy_lang"]`.

```tsx
import { LanguageSwitcher } from "@/components/LanguageSwitcher";

// Inserisci dove serve il selettore lingua
<LanguageSwitcher />
```

## Testare il cambio lingua localmente

1. Avvia lo stack dev o deploy (vedi `docs/guides/LOCAL_DOCKER_START.md`)
2. Apri l'app nel browser
3. Clicca 🇬🇧 nella navbar → l'interfaccia passa in inglese
4. Ricarica la pagina → la lingua è mantenuta (via `localStorage["andy_lang"]`)
5. Clicca 🇮🇹 per tornare in italiano

Per forzare una lingua via console browser:

```js
localStorage.setItem("andy_lang", "en");
location.reload();
```

## Aggiungere una nuova lingua

1. Creare `apps/web/i18n/<codice>.json` (es. `fr.json`) con tutte le chiavi
2. In `apps/web/lib/i18n.ts` aggiungere il codice all'array `supportedLngs`
3. Aggiungere il pulsante in `LanguageSwitcher.tsx`
4. Ricostruire lo stack: `npm run docker:test:nocache`
